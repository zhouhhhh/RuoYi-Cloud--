# README 设计

## 目标

在仓库根目录新增 `README.md`，作为学习者重新进入项目时的快速入口。
README 负责回答“这是什么、当前做到哪里、如何启动、详细文档在哪里”，
不复制 `AGENTS.md` 中的完整学习规则和 16 个里程碑验收细节。

## 目标读者

主要读者是项目学习者本人。README 应支持在较长时间未打开项目后，快速恢复
项目上下文并进入当前任务。

## 内容结构

README 按以下顺序组织：

1. 项目名称和一句话定位。
2. 项目目标与当前进度。
3. 精简技术栈及关键版本。
4. 后端、前端、基础设施和文档目录。
5. 快速启动，包括基础设施、后端和前端命令。
6. 常用访问地址。
7. 六阶段学习路线摘要。
8. `AGENTS.md`、教学模式和审查模式的文档入口。
9. 数据库初始化、Nacos 凭据和 Docker 数据卷注意事项。

## 信息边界

- README 保持在约 100 至 150 行。
- 当前里程碑固定为 `01 环境基线`，并链接到 `AGENTS.md`。
- 学习路线只展示六个阶段，不展开 16 个任务的完成标准。
- 启动命令必须标注执行目录。
- 版本、端口、目录和命令以仓库现有配置为准。
- 不使用徽章、截图或营销性内容。
- 不记录密码、令牌、本机绝对路径或临时运行结果。

## 快速启动流程

入口流程为：

```text
检查环境
-> 启动 MySQL 和 Redis
-> 初始化 ry-cloud 与 ry-config
-> 启动 Nacos
-> 启动 Gateway、Auth、System、Gen
-> 启动 Vue3 前端
-> 访问前端并登录
```

README 给出常用命令，但数据库首次初始化的详细操作仍由里程碑 01 分步指导，
避免在入口文档中混入环境相关密码。

## 错误处理提示

README 只保留三个高频提醒：

- 首次启动必须先初始化数据库，再启动依赖 `ry-config` 的 Nacos。
- Nacos 中的 MySQL、Redis 配置必须与 `docker-compose.local.yml` 一致。
- 停止 Compose 时不要默认使用 `-v`，避免删除学习数据。

完整排错顺序链接到 `AGENTS.md`。

## 验证

创建 README 后执行以下验证：

- 根目录 `README.md` 存在。
- README 中引用的相对路径全部存在。
- Maven、npm 和 Docker Compose 命令与仓库配置一致。
- Vue3、Gateway、Nacos 等主要访问端口与配置一致。
- 文档不含通用占位符和敏感信息。
- `git diff --check` 无格式错误。
